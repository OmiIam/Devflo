# Security Policy for DevFlo

## Reporting a Vulnerability
If you discover a security issue in DevFlo, please report it privately. Use GitHub's private vulnerability reporting feature linked below:

`https://github.com/OmiIam/Devflo/security/policy`

Do not open an issue in the public repository — this helps us protect other users while we fix the problem.
What to Include in Your Report
To help us understand and fix the issue quickly, include:
- A description of what you found
- Steps to reproduce it (what you did, what files or commands you used)
- Any error messages or logs you observed
- The version of DevFlo you were using
- Screenshots or sample files if it helps explain the problem

Don’t worry if you’re not a security expert — just explain what happened in simple words.

## What to Expect After Reporting
- We will acknowledge your report within 48 hours.
- We may contact you for more details if needed.
- Once the issue is fixed, we will credit you in release notes if you want.
- Fixes will be released in the next update, and we will let you know when it’s available.

We take security seriously and appreciate your help keeping DevFlo safe for everyone.
